# Competitor Analysis

Research and analyze a competitor or competitive category relevant to Kinroot.

## Variables

- target: $ARGUMENTS (required — competitor name, category, or comparison topic)

## Instructions

### Step 1: Context

Read these files for Kinroot positioning:
- context/business-info.md — what Kinroot is and is not
- context/strategy.md — current priorities
- reference/competitive-landscape.md — existing competitive intel
- reference/brand-voice.md — positioning language

### Step 2: Research

Search the web for current information on the target. Focus on:
- **Product:** What do they offer? Core value prop?
- **Pricing:** Free tier? Paid tiers? What is included at each level?
- **Users:** Who uses it? How many? What do reviews say?
- **Strengths:** What do they do well? What do users love?
- **Weaknesses:** What do users complain about? Where do they fall short?
- **Recent changes:** Any new features, pivots, or funding in the last 6 months?

### Step 3: Analyze Against Kinroot

For each finding, map it to Kinroot:
1. **Where Kinroot wins** — specific advantages we have
2. **Where they win** — specific advantages they have
3. **Opportunities** — gaps we could exploit or features we should prioritize
4. **Threats** — things they do that could pull our target users away
5. **Positioning implications** — how should we talk about Kinroot differently given this?

### Step 4: Output

Save to: outputs/competitor-analysis-YYYY-MM-DD-{target-slug}.md

Structure:
1. **Executive Summary** (3-5 sentences)
2. **Competitor Profile** (product, pricing, users, positioning)
3. **SWOT vs Kinroot** (strengths, weaknesses, opportunities, threats)
4. **Feature Comparison Table** (side-by-side)
5. **Strategic Recommendations** (what should Kinroot do?)
6. **Sources**

### Step 5: Update Reference

If significant new intel was discovered, suggest specific updates to reference/competitive-landscape.md.

### Step 6: Summary

Provide the 3 most actionable takeaways verbally.
